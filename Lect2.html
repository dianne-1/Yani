1. The Professional's Toolkit

Before building, professionals prepare their environment:

  • Hardware: A computer (any specs work, but investment is recommended for serious work).
  • Text Editor: Notepad++ or similar code editors.
  • Browsers: Chrome, Firefox, Edge, or Safari for testing.
  • Project Structure: A main folder containing index.html, and subfolders for images, styles, and scripts .

2. What is HTML5?

HTML (Hypertext Markup Language) is a markup language, not a programming language.

  • Timeline: First published in 1991; HTML5 was officially published in 2012.
  • Goal: HTML5 aims to reduce the need for external plugins, support existing web pages, and improve the "semantic" (meaning) of elements.

3. Basic Document Structure

A standard HTML5 document follows this hierarchy:

  • <!DOCTYPE html>: Defines the document as HTML5.
  • <html lang="en">: The root element wrapping all content.
  • <head>: Container for metadata, title, and links (not visible to viewers).
  • <body>: Contains all visible content like text and images.

4. Semantic Elements

HTML5 introduced elements that describe their meaning to both the browser and developer:

  • <header>: Introductory content.
  • <nav>: Major blocks of navigation links.
  • <section>: A thematic grouping of content.
  • <article>: Independent, self-contained content (e.g., blog posts).
  • <aside>: Content related to the main topic but placed aside (sidebars).
  • <footer>: Information about the author, copyright, or contact info.

5. Text Controls & Formatting

Elements used to define the look and function of text:

  • Headings: <h1> (largest) to <h6> (smallest).
  • Strong/Bold: <strong> (logical emphasis) and <b> (physical bold) .
  • Emphasis: <em> (logical) and <i> (physical italics).
  • Line Breaks: <br> (next line) vs <p> (adds a blank line).
  • Entities: Special characters like &nbsp; (space), &lt; (<), and &copy; (©).
